Các dạng tài liệu mà dân công nghệ có thể bạn sẽ gặp phải khi học/làm nghề IT:

1. Tài liệu yêu cầu

- BRD - Tài liệu yêu cầu nghiệp vụ
- SRS - Tài liệu đặc tả yêu cầu phần mềm
- User Story - Mô tả nhu cầu dưới góc nhìn người dùng
- Use Case - Mô tả các tình huống sử dụng hệ thống
- Business Rule - Quy tắc nghiệp vụ

2. Tài liệu thiết kế hệ thống

- System Design Document - Tài liệu thiết kế hệ thống
- Architecture Diagram - Sơ đồ kiến trúc
- Database Design - Thiết kế cơ sở dữ liệu
- ERD - Sơ đồ quan hệ dữ liệu
- API Specification - Tài liệu mô tả API
- Sequence Diagram - Sơ đồ luồng xử lý theo thời gian
- Flowchart - Sơ đồ luồng nghiệp vụ / luồng xử lý

3. Tài liệu kiểm thử

- Test Plan - Kế hoạch kiểm thử
- Test Scenario - Kịch bản kiểm thử
- Test Case - Trường hợp kiểm thử chi tiết
- Bug Report - Báo cáo lỗi
- UAT Document - Tài liệu nghiệm thu người dùng

4. Tài liệu triển khai và vận hành

- Deployment Guide - Hướng dẫn triển khai
- Release Note - Ghi chú phiên bản mới
- Runbook - Hướng dẫn xử lý sự cố
- Monitoring Document - Tài liệu theo dõi hệ thống
- Backup / Recovery Plan - Kế hoạch sao lưu và khôi phục

5. Tài liệu hướng dẫn sử dụng

- User Manual - Hướng dẫn sử dụng cho người dùng
- Admin Guide - Hướng dẫn cho quản trị viên
- FAQ - Câu hỏi thường gặp
- Training Material - Tài liệu đào tạo người dùng

6. Tài liệu quản lý dự án

- Project Plan - Kế hoạch dự án
- Timeline / Roadmap - Lộ trình phát triển
- Meeting Minutes - Biên bản họp
- Risk Log - Danh sách rủi ro
- Change Request - Yêu cầu thay đổi
- Scope Document - Tài liệu phạm vi dự án

7. Tài liệu dữ liệu và báo cáo

- Data Dictionary - Từ điển dữ liệu
- Metric Definition - Định nghĩa chỉ số
- Dashboard Requirement - Yêu cầu dashboard
- Report Specification - Đặc tả báo cáo
- Analysis Report - Báo cáo phân tích
- Data Flow Diagram - Sơ đồ luồng dữ liệu

8. Tài liệu sản phẩm

- Product Requirement Document - Tài liệu yêu cầu sản phẩm
- Product Roadmap - Lộ trình sản phẩm
- Feature Specification - Đặc tả tính năng
- User Flow - Luồng người dùng
- Wireframe / Prototype - Bản phác thảo giao diện

9. Tài liệu bảo mật và phân quyền

- Access Control Matrix - Ma trận phân quyền
- Security Requirement - Yêu cầu bảo mật
- Privacy Policy - Chính sách quyền riêng tư
- Audit Log Document - Tài liệu ghi nhận lịch sử thao tác

10. Tài liệu bàn giao

- Handover Document - Tài liệu bàn giao
- Technical Documentation - Tài liệu kỹ thuật
- Source Code Guideline - Quy chuẩn code
- Environment Information - Thông tin môi trường
- Maintenance Guide - Hướng dẫn bảo trì
  Làm công nghệ không chỉ là viết code. Bạn sẽ phải đọc, viết, trao đổi và làm việc với rất nhiều loại tài liệu khác nhau.
  Vì tài liệu chính là thứ giúp BA, Dev, Tester, Data, PM, khách hàng và người dùng hiểu cùng một vấn đề theo cùng một cách.
